# Trading Psychology Panel — 交易心理面板

## 概述

为交易日志系统新增心理框架功能，帮助交易者重塑对止损和踏空的认知：

1. **止损成本化** — 把止损视为"购买一张概率彩票的票价"，开仓时记录风险金额作为心理锚定
2. **踏空记录** — 记录看到信号但未入场的情况，强化"踏空的利润从来不属于你"的心态

## 核心理念

- 止损 = 买彩票的票价，是做交易的成本，不是"亏损"
- 开仓 = 购买一张有正期望值的彩票（基于历史胜率）
- 踏空 = 没买的彩票，中不中奖跟你无关

## 数据模型变更

### trades 表

**新增字段：**
- `risk_amount` REAL DEFAULT NULL — 本单风险金额（美元），开仓时用户手动填入

**status 字段扩展：**
- 现有值：`open`（持仓中）、`closed`（已平仓）
- 新增值：`missed`（踏空/未入场）

**missed 状态记录的字段要求：**
- 必填：`date`, `pair`, `direction`, `user_id`
- 可选：`notes`（为什么没入场）
- 其余字段（lots, entry, stop, target, exit_price, gross_pnl 等）留 NULL

### 数据库迁移

```sql
ALTER TABLE trades ADD COLUMN risk_amount REAL DEFAULT NULL;
```

status 字段为 TEXT 类型，无需迁移，直接支持新值 `missed`。

## API 变更

### POST /api/trades

- 接受新字段 `risk_amount`（可选）
- 接受 `status: 'missed'`，此时只校验 date, pair, direction

### PUT /api/trades/:id

- 支持更新 `risk_amount`

### GET /api/trades

- 返回所有状态的记录（包括 missed）
- 前端按 status 过滤显示

## 前端变更

### 1. 开仓表单（TradeForm.jsx）

- 在 `openOnly=true` 模式下，止损价字段后新增"本单风险($)"输入框
- 字段名 `risk_amount`，类型 number，非必填
- 随表单一起提交

### 2. 心理面板（PsychologyPanel — 新组件）

**位置：** Record tab 顶部，开仓表单上方

**3 个 KPI 卡片：**

| 卡片 | 数据来源 | 主数字 | 副标题 |
|------|---------|--------|--------|
| 今日成本 | 今日 open 状态交易的 risk_amount 总和 | $XXX | X 张彩票 |
| 历史胜率 | calcStats().winRate | XX% | 基于 N 笔交易 |
| 今日踏空 | 今日 missed 状态记录数 | N | 与你无关的利润 |

**踏空录入：**
- 踏空卡片右上角"+"按钮
- 点击后内联展开极简表单：日期（默认今天）、品种（下拉）、方向（多/空）、备注（文本框）、"记录"按钮
- 提交后创建 `status=missed` 的 trade 记录，面板计数实时更新

**踏空历史：**
- 点击踏空计数数字可展开/折叠今日踏空记录列表
- 每条显示：品种、方向、备注
- 可删除

### 3. 交易列表（TradeTable.jsx）

- 过滤掉 `status=missed` 的记录，不在主表格中显示
- missed 记录只在心理面板中可见

### 4. 持仓列表（OpenPositions.jsx）

- 无变更，仍只显示 `status=open`

### 5. 计算逻辑（calc.js）

- `calcStats()` 继续只统计 `closed` 状态交易（现有逻辑已天然兼容，只对有 exit_price 的记录计算）
- 新增辅助函数或在面板组件内计算：
  - `todayRiskTotal`：今日 open 交易的 risk_amount 总和
  - `todayTickets`：今日 open 交易笔数
  - `todayMissedCount`：今日 missed 记录数

### 6. 导出/导入

- 导出包含 `risk_amount` 字段和 `missed` 状态记录
- 导入兼容这些新字段和状态

## UI 设计

### 心理面板布局

```
┌─────────────┬─────────────┬─────────────┐
│  今日成本    │  历史胜率    │  今日踏空 [+]│
│   $450      │    62%      │     2       │
│  3 张彩票   │ 基于147笔交易│ 与你无关的利润│
└─────────────┴─────────────┴─────────────┘
┌─ 踏空表单（展开态）──────────────────────┐
│ [日期] [品种 ▾] [方向 ▾] [备注...] [记录]│
└─────────────────────────────────────────┘
```

### 样式

- 复用现有 KpiCard 组件
- 配色：成本=蓝色、胜率=绿色、踏空=黄色
- 暗色主题兼容，跟随系统主题切换

## 不做的事情

- 不在 stats 页面新增踏空相关统计（保持聚焦）
- 不在开仓确认时弹窗（用户选择了实时展示而非弹窗）
- 不自动计算风险金额（用户手动输入，心理锚定效果更强）
- 不记录踏空的详细价格/止损/目标（极简原则）
- 不按品种/策略分别计算胜率（使用整体胜率）
